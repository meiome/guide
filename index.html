git add — differenze tra -u, . e -A

I tre flag sembrano equivalenti ma si comportano in modo diverso rispetto a file modificati, nuovi ed eliminati, e rispetto alla cartella corrente.

Comando File modificati File nuovi File eliminati Ambito
git add -u Sì No Sì Tutto il repo
git add . Sì Sì Sì (da Git 2.0) Solo dalla cartella corrente in giù
git add -A Sì Sì Sì Tutto il repo

In pratica:

  • -u (update): aggiorna solo i file già tracciati. Utile quando non vuoi aggiungere file nuovi.
  • .: include tutto a partire dalla cartella in cui ti trovi. Se sei in una sottocartella, file modificati altrove restano fuori.
  • -A (all): include tutto il repo. È l'opzione più "totale".

Generatore commit

git add -A && git commit -m ""

I caratteri ", `, $, \ vengono automaticamente escapati per funzionare in shell.
